📊 現状分析レポート
✅ 必要な機能 (Core Features)
これらはアプリの核となる価値であり、維持・強化すべきです。

歌詞同期プレイヤー (LyricSyncPlayer): 「聞き流し・インプット」用として機能しており必須です。
学習進行ページ (LyricProgressPage): 「精読・アウトプット」用として、Study -> Test -> Puzzle -> Reward のサイクルは学習効果が高く、非常に良い設計です。
単語帳 (LibraryPage): 学習した単語の保存場所として必須です。
永続化 (Persistence): zustand/persist によるローカル保存は実装済みで、MVPとしては十分機能しています。
⚠️ 不足している機能 (Missing / Critical)
アプリとして完成度を高めるために優先的に実装すべき機能です。

複数曲のサポート (最重要):
現状は song_data.json が単一オブジェクトで、コード内でIDもハードコードされています。
必要なもの: 曲一覧画面、曲データ構造の配列化、動的なデータ読み込み。
ルーティング (react-router-dom):
現状の useState による画面切り替えは、ブラウザの「戻る」ボタンが効かず、特定の曲や画面へのリンクもできません。
将来的に曲が増えた際、/song/twinkle-star のようなURLアクセスが必須になります。
本格的な学習アルゴリズム (SRS):
現状の「習熟度トグル (0->1->2)」は単純すぎます。忘却曲線に基づいた復習タイミングの提示があると、学習アプリとしての価値が跳ね上がります。
✂️ 余分・整理可能な機能 (Redundant / Refactor)
独立した「速読トレーナー」:
LyricQuizPage と機能が重複気味です。クイズモードの一つとして統合することで、メニューをシンプルにできます。
現在のダッシュボード:
単なるメニュー画面になっています。「曲一覧」をメイン画面（ホーム）にし、そこから各学習モードに入れるようにすると、UXが向上します。
