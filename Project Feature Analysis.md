📊 現状分析レポート
⚠️ 不足している機能 (Missing / Critical)
アプリとして完成度を高めるために優先的に実装すべき機能です。

複数曲のサポート:
現状は song_data.json が単一オブジェクトで、コード内でIDもハードコードされています。
必要なもの: 曲一覧画面、曲データ構造の配列化、動的なデータ読み込み。

ルーティング (react-router-dom):
現状の useState による画面切り替えは、ブラウザの「戻る」ボタンが効かず、特定の曲や画面へのリンクもできません。
将来的に曲が増えた際、/song/twinkle-star のようなURLアクセスが必須になります。

本格的な学習アルゴリズム (SRS):
現状の「習熟度トグル (0->1->2)」は単純すぎます。忘却曲線に基づいた復習タイミングの提示があると、学習アプリとしての価値が跳ね上がります。

✂️ 整理可能な機能 (Refactor)

現在のダッシュボード:
単なるメニュー画面になっています。「曲一覧」をメイン画面（ホーム）にし、そこから各学習モードに入れるようにすると、UXが向上します。

✅ リファクタリングのロードマップ
現在のコードベースは「機能の詰め込みすぎ」でUXが分散しています。以下のように整理することを推奨します。

拡張（Expansion）: song_data.json の構造を変え、曲一覧画面（Home）を作成する。

統合（Cleanup）: アプリの導線を「ダッシュボード → 歌詞進行学習（Progress）」に集中させる。
